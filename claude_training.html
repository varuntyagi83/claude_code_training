Jargon Buster: Technical Terms Explained

Let's demystify some terms you might encounter:

üî§ Jargon Buster
Electron

This is the technology VS Code is built with. It allows developers to create desktop apps using web technologies.

üëâ Simple version: Electron is just the "recipe" used to make VS Code. You'll never interact with it directly ‚Äî it's like knowing your car has an engine without needing to understand how engines work.
üî§ Jargon Buster
Extension

A small add-on that gives VS Code new powers, like installing an app on your phone.

üëâ Simple version: Extensions are like phone apps. Claude Code is an extension you "install" to add AI powers to VS Code.
üî§ Jargon Buster
Terminal / CLI

A text-based way to give commands to your computer. Instead of clicking, you type instructions.

üëâ Simple version: It's like texting your computer instead of pointing and clicking. You type "claude" and press Enter to start Claude Code.
üî§ Jargon Buster
MCP (Model Context Protocol)

A system that lets Claude Code connect to other apps and services.

üëâ Simple version: MCP is like a universal adapter/translator that lets Claude "talk" to other apps like ClickUp, n8n, or databases.
üî§ Jargon Buster
API (Application Programming Interface)

A way for different software programs to communicate with each other.

üëâ Simple version: An API is like a waiter in a restaurant ‚Äî it takes your order (request), brings it to the kitchen (another app), and returns with your food (the response).
üî§ Jargon Buster
LSP (Language Server Protocol)

Technology that gives VS Code smart features like auto-complete and error detection.

üëâ Simple version: LSP is what makes VS Code "smart" ‚Äî it's why the app can suggest words as you type, like your phone's keyboard does.
